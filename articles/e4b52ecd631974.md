## [ABC086A - Product](https://atcoder.jp/contests/abs/tasks/abc086_a)

問題概要: 二つの正整数`a`、`b`が与えられるので、`a`と`b`の積が偶数か奇数かを判定してください。

### 下準備

最初にいくつか説明するべきことがあります。

#### 標準入出力

標準出力は組み込みの`println`を使えばよいのですが、標準入力用の関数はないのでNode.js用のFFIを書く必要があります。

https://www.moonbitlang.com/pearls/moonbit-jsffi#first-javascript-api-call

JS FFIは次のように書きます。


標準入力は次のように書けます。`"extern : "js"`はFFI宣言のキーワード、その次が関数の型宣言、最後は関数の中身の文字列です。`#|`はMoonbitでの複数行の文字列リテラルです。

```mbt
extern "js" fn read_stdin() -> String =
  #| function() {
  #|   const fs = require("fs");
  #|   return fs.readFileSync(0, "utf8");
  #| }
```

#### 文字列->Intへの変換

標準ライブラリの`@strconv`パッケージの、`parse_int`や`from_str`が使えます。パッケージを使うには`moon.pkg`でインポートする必要があります。

```diff text:moon.pkg
import {
+  "moonbitlang/core/strconv",
}

options(
  "is-main": true,
)
```
コードからは、パッケージのパスの最後のセグメントに`@`をつけたもの（ここでは`@strconv`）でアクセスします。

```mbt
fn to_int(x : StringView) -> Int {
  try! @strconv.parse_int(x)
}
```

`@strconv.parse_int`は変換に失敗した場合は例外をスローするのですが、整数だと分かっている場合にのみ使うので`try!`で握り潰します。

#### パターンマッチを使った入力のパース

例えば、整数`a`、`b`を受け取る場合は以下のように書けます。

```mbt
match input {
  [a, b] => {
    // ここにプログラムを書く
  }
  _ => {
    // ここは入力が不正な場合(AtCoderでは基本的に実行されない)
    panic()
  }
}
```

不変条件チェックのための`guard`構文を使うと簡潔に書けます。

```mbt
guard input is [a, b]
```

https://docs.moonbitlang.com/en/latest/language/fundamentals.html#guard-statement

### 解答

必要な道具は揃ったので、あとは実装しましょう！

```mbt
///|
extern "js" fn read_stdin() -> String =
  #| function() {
  #|   const fs = require("fs");
  #|   return fs.readFileSync(0, "utf8");
  #| }

///|
fn main {
  let input = read_stdin()
  guard input.trim().split(" ").to_array().map(x => try! @strconv.parse_int(x))
    is [a, b]
  let ans = if a * b % 2 == 0 { "Even" } else { "Odd" }
  println(ans)
}
```

`String::split`で得られるのはイテレーターなので、パターンマッチするために`to_array`で配列に変換しています。

また、ifは式なので変数にそのまま代入できます。

## [ABC081A - Placing Marbles](https://atcoder.jp/contests/abs/tasks/abc081_a)

問題概要: `0`または`1`からなる3桁の文字列が与えられるので、`1`の個数を数えてください。

### 解答

文字列をループしてカウントします。ループには`for-in`や、古典的な`for init; condition; increment`が使えます。今回は`for-in`で十分です。

```mbt
///|
fn main {
  let s = read_stdin().trim()

  let mut ans = 0
  for c in s {
    if c == '1' {
      ans += 1
    }
  }
  println(ans)
}
```

[提出 #73560224 - AtCoder Beginners Selection](https://atcoder.jp/contests/abs/submissions/73560224)

### 別解

```mbt
///|
fn main {
  let s = read_stdin().trim()
  let ans = s.iter().filter(c => c == '1').count()
  println(ans)
}
```

## [ABC081B - Shift only](https://atcoder.jp/contests/abs/tasks/abc081_b)

問題概要: N個の整数`A_1`, ... , `A_N`があります。すべての整数が偶数のとき、全てを2で割ったものに置き換える操作が行える。最大で何回操作が行えるか求めてください。

### 解答

2で割れる回数の最小値を求めればよいです。素直にループを書きました。

[提出 #73560496 - AtCoder Beginners Selection](https://atcoder.jp/contests/abs/submissions/73560496)

```mbt
///|
fn main {
  let input = read_stdin()
  guard input.trim().split("\n").to_array() is [_n, a]
  let a = a.split(" ").map(to_int).to_array()

  let mut ans = 30
  for num in a {
    let mut count = 0
    let mut num = num
    while num % 2 == 0 {
      num /= 2
      count += 1
    }
    ans = Int::min(ans, count)
  }
  println(ans)
}
```

未使用の変数の警告は、`_`を先頭につけると消せます。`_n`としていますが`_`だけでもOKです。

### 別解

`map`と`fold`で宣言的に書いてみました。

```mbt
///|
fn div_count(num : Int) -> Int {
  let mut num = num
  let mut count = 0
  while num % 2 == 0 {
    num /= 2
    count += 1
  }
  count
}

///|
fn main {
  let input = read_stdin()
  guard input.trim().split("\n").to_array() is [_n, a]
  let a = a.split(" ").map(to_int)
  let ans = a.map(div_count).fold(init=30, Int::min)
  println(ans)
}
```

## [ABC087B - Coins](https://atcoder.jp/contests/abs/tasks/abc087_b)

問題概要: 500円玉が`A`枚、100円玉が`B`枚、50円玉が`C`枚あります。これらの中から何枚か選んで、合計金額を`X`円にする方法が何通りあるか数えてください。

### 解答

3重ループで全探索します。`for-in`ループではRange表記が使えます。半開区間は`a..<b`、閉区間は`a..<=b`です。

https://docs.moonbitlang.com/en/latest/language/fundamentals.html#range-expression-in-for-in-loop

[提出 #73561308 - AtCoder Beginners Selection](https://atcoder.jp/contests/abs/submissions/73561308)

```mbt
///|
fn main {
  let input = read_stdin()
  guard input.trim().split("\n").map(to_int).to_array() is [a, b, c, x]

  let mut ans = 0
  for i in 0..<=a {
    for j in 0..<=b {
      for k in 0..<=c {
        let sum = 500 * i + 100 * j + 50 * k
        if sum == x {
          ans += 1
        }
      }
    }
  }
  println(ans)
}
```

なお、降順ループは`b>..a`や`b>=..a`で書けます。直感的ですね。

## [ABC083B - Some Sums](https://atcoder.jp/contests/abs/tasks/abc083_b)

問題概要: 1以上N以下の整数のうち、10進法での各桁の和がA以上B以下であるものの総和を求めてください。

### 解答

各桁の和を求める`digit_sum`関数を作り、あとは書かれている通りに実装しました。

`digit_sum`関数のテストも書いてみました。`moon test [path]`で実行できます。適宜デバッグしながら実装できるのは嬉しいですね。テストについては以下を参照してください。

https://docs.moonbitlang.com/en/latest/language/tests.html

```mbt
///|
fn digit_sum(x : Int) -> Int {
  // 各桁の和を求める
  let mut x = x
  let mut sum = 0
  while x > 0 {
    sum += x % 10
    x /= 10
  }
  sum
}

///|
test "digit_sum" {
  assert_eq(digit_sum(23), 5)
  assert_eq(digit_sum(3141), 9)
}

///|
fn main {
  let input = read_stdin()
  guard input.trim().split(" ").map(to_int).to_array() is [n, a, b]

  let mut ans = 0
  for i in 1..<=n {
    let sum = digit_sum(i)
    if a <= sum && sum <= b {
      ans += i
    }
  }
  println(ans)
}
```

## [ABC088B - Card Game for Two](https://atcoder.jp/contests/abs/tasks/abc088_b)

問題概要: N枚のカードがあり、i枚目のカードには数a_iが書かれています。AliceとBobが、大きい数字が書かれたカードを交互に取っていきます。AliceとBobの得点差がいくつになるか求めてください。

### 解答

降順ソートをして順番に足していきます。ソートについて`moon doc Array`を実行すると、`Array::sort`、`Array::sort_by`、`Array::sort_by_key`などの関数があるのがわかります。

```text
$ moon doc Array | grep sort
  pub fn[T : Compare] Array::is_sorted(Self[T]) -> Bool
  pub fn[T : Compare] Array::sort(Self[T]) -> Unit
  pub fn[T] Array::sort_by(Self[T], (T, T) -> Int) -> Unit
  pub fn[T, K : Compare] Array::sort_by_key(Self[T], (T) -> K) -> Unit
```

reverseする関数は`Array::rev`がありました。

また、関数の戻り値を見ると、ソート関数は`Unit`ですが、`Array::rev`は`Self[T]`でした。つまり、ソートは破壊的な一方、`Array::rev`は非破壊的っぽいことがわかります。実際、コードジャンプで関数のコメントを見てみるとそうだったので、注意して実装します。

```mbt
///|
fn main {
  let input = read_stdin()
  guard input.trim().split("\n").to_array() is [_n, a]
  let a = a.split(" ").map(to_int).to_array()
  a.sort()
  let a = a.rev()

  let mut ans = 0
  for i, num in a {
    ans += if i % 2 == 0 { num } else { -num }
  }
  println(ans)
}
```

## [ABC085C - Otoshidama](https://atcoder.jp/contests/abs/tasks/abc085_c)

問題概要: 10000円札、5000円札、1000円札が計N枚あります。合計Y円になる組み合わせを一つ出力してください。

### 解答

N <= 2000なので、10000円札の枚数と5000円札の枚数をループして、O(N^2)で解きます。今回はループの条件がやや複雑なので、古典的なforループで書きました。

```mbt
///|
fn solve(n : Int, y : Int) -> (Int, Int, Int) {
  for i = 0; i <= n; i = i + 1 {
    for j = 0; i + j <= n; j = j + 1 {
      let k = n - (i + j)
      let sum = 10000 * i + 5000 * j + 1000 * k
      if sum == y {
        return (i, j, k)
      }
    }
  }
  return (-1, -1, -1)
}

///|
fn main {
  guard read_stdin().trim().split(" ").map(to_int).to_array() is [n, y]
  let ans = solve(n, y)
  println("\{ans.0} \{ans.1} \{ans.2}")
}
```

## [ABC049C - 白昼夢](https://atcoder.jp/contests/abs/tasks/arc065_a)

問題概要: 文字列Sが与えられます。`dream`、`dreamer`、`erase`、`eraser`の4つの文字列を組み合わせてSを作れるかどうか判定してください。

### 解答

`dp[i] := i番目の文字まで一致させることができるか` としてDPをします。

遷移ができるのは、`dp[i] = true`かつ、`s`の`i`番目から`i + |t|`番目の文字が`t`(`= dream | dreamer | erase | eraser`)に一致する場合です。

文字列のスライスは`s[start:end]`のように書きます。`String::sub`も同じ意味の関数です。

範囲外参照のために`CreatingViewError`をスローするようになっているので、よしなにハンドリングします。

```mbt
///|
fn main {
  let s = read_stdin().trim()
  let n = s.length()
  let ts = ["dream", "dreamer", "erase", "eraser"]

  let dp = Array::make(n + 1, false)
  dp[0] = true
  for i in 0..<n {
    if dp[i] {
      for t in ts {
        // sのi文字目以降が、tから始まっていればOK
        if i + t.length() > n {
          continue
        }
        let slice = s[i:i + t.length()] catch { _ => "" }
        if slice == t {
          dp[i + t.length()] = true
        }
      }
    }
  }
  println(if dp[n] { "YES" } else { "NO" })
}
```

[提出 #73579962 - AtCoder Beginners Selection](https://atcoder.jp/contests/abs/submissions/73579962)

`StringView::has_prefix`を使うと簡潔に書けます。スライスで範囲外参照をしないことが明らかなので`try!`で握りつぶしました。

```mbt
///|
fn main {
  let s = read_stdin().trim()
  let n = s.length()
  let ts = ["dream", "dreamer", "erase", "eraser"]

  let dp = Array::make(n + 1, false)
  dp[0] = true
  for i in 0..<n {
    if !dp[i] {
      continue
    }
    for t in ts {
      let ok = (try! s[i:]).has_prefix(t)
      if ok {
        dp[i + t.length()] = true
      }
    }
  }
  println(if dp[n] { "YES" } else { "NO" })
}
```

## [ABC086C - Traveling](https://atcoder.jp/contests/abs/tasks/arc089_a)

問題概要: 2次元平面上を、時刻`0`に`(0, 0)`を出発して、時刻`t_i`に`(x_i, y_i)`を訪れる予定があります。時刻1につき上下左右に移動できますが、その場にとどまることはできません。全ての予定を実行可能か判定してください。

### 解答

次の点に時間通りに移動できるのは、移動するのに十分に時間があり、かつ移動時間と移動距離の偶奇が一致している場合です。全ての移動についてその条件を満たしているかを確認すればOKです。

予定を保存するために`Plan`構造体を作りました。

```mbt
///|
struct Plan {
  t : Int
  x : Int
  y : Int
}

///|
fn main {
  let input = read_stdin()
  guard input.trim().split("\n").to_array() is [_, .. plans]
  let plans = plans.map(plan => {
    guard plan.split(" ").map(to_int).to_array() is [t, x, y]
    Plan::{ t, x, y }
  })

  let mut now = { t: 0, x: 0, y: 0 }
  for plan in plans {
    let dt = plan.t - now.t
    let dx = Int::abs(plan.x - now.x)
    let dy = Int::abs(plan.y - now.y)

    if dt < dx + dy || dt % 2 != (dx + dy) % 2 {
      println("No")
      return
    }
    now = plan
  }
  println("Yes")
}
```

構造体を組み立てるには`構造体名::{ field: fieldの値 }`のようにします。JavaScriptと同じく省略記法があります。また、`構造体名::`は省略でき、その場合はフィールド名が一致している構造体に型推論されます。

このコードでは使っていませんが、構造体のスプレッド構文（`{ ..point, x: 1 }`など）もあります。
